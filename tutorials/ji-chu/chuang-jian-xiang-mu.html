创建项目

使用 VD 创建一个新项目有三种姿势：

  • 拖拽一个空文件夹到“打开项目面板”
  • 单击“打开项目面板”上的图标，选择一个空文件夹
  • 在空文件夹上或内右键，选择“Open With VD” 打开

不管以哪种姿势打开，都会进入创建项目页。

VD 期望包含齐全的项目类型，现已实现的项目类型有：

  • [x] PC Web 端
  • [x] 移动 Web 端
  • [x] 库
  • [x] PC 客户端
  • [ ] 移动客户端

其他项目字段信息及说明如下：

字段 说明
项目名称 项目名称， 一旦定下来就不允许修改，比如“VD”
产品名称 项目对外的称呼，后续可以需改，比如 VD 项目的产品名称“前端可视化开发平台”
项目描述 争对项目的简短描述或产品标语，比如 VD 项目的产品标语“前端项目源码管理平台”
版本号 项目的版本号，格式参考： 语义化版本
兼容性 项目兼容性，包含了主流浏览器内核所代表的浏览器：Chrome（Blink）、Firefox（Gecko）、Safari（Webkit）、IE（Trident）
路由类型 单页应用部署的路由方式，库类型没有此项；可选值：BrowserRouter、HashRouter，默认值为 BrowserRouter
参与人员 应用研发中各环节参与角色人员，产品、设计、前端、后端、测试、运维；便于沟通、协调
UI 版本库地址 版本库地址，存储 UI 视觉稿文件
代码版本库地址 版本库地址，存储前端源码
BUG 管理平台地址 应用测试反馈平台地址，比如：ERP、第三方 BUG 管理平台
联调环境地址 服务端环境地址，供前后端功能联调使用
预发布环境地址 服务端环境地址，供产品灰度上线使用
线上环境地址 服务端环境地址，正式线上环境
支持换肤 应用是否支持更换皮肤，默认值为，false
支持跨终端（响应式） 应用是否支持响应式设计，默认值为 false
支持平台 应用支持的平台，只有客户端类型才有此项；可选值有 macOS、Windows、Linux，默认值为 Windows
其他说明 可以记录一项目变更记录，更新日志等等其他信息
