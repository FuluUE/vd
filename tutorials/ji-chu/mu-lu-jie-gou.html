目录结构

目录结构

使用 VD 创建项目会生成一些项目样板，VD 将源码环境分为：开发环境、构建中转环境、生产环境；根据项目类型的不同，生产环境的目录结构会有所不同。

src          // 开发环境目录
|- components                // 展示组件目录
|    |- less                 // 样式资源目录
|    |- images               // 图片资源目录
|    |- json                 // 静态数据目录
|    |- doc                  // 使用文档目录
|    |- __test__             // 单元测试用例目录
|    |- 组件名（eg: Icon.js） // 组件源码文件
|    `- index.js             // 组件入口文件
|- containers               // 容器组件目录
|- redux               // 异步流目录
|    |- reducers.js       // Reducer（rootReducer）配置文件
|    `- sagas.js          // Saga（rootSaga） 配置文件
|- store                  // Store 目录
|    `- configureStore.js // Store 配置文件
|- utils                  // 工具集目录
|    `- axios.js          // 请求及全局异常处理配置文件
|- configs                // 配置文件目录
|    `- api.js            // API 接口地址配置对象
|- sprite                 // CSS Sprite 源图目录
.build        // 构建中转环境目录
dist          // 生产环境目录
    `- resources             // 资源目录
        |- dll               // 脚本资源目录
        |- js                // DLL 资源目录
        |- css               // 样式资源目录
        `- images            // 图片资源目录
App.js        // 单页应用入口组件
index.js      // 单页应用入口文件
index.ejs     // HTML 模板文件

在 VD@1.4.x 以后，内置了 dva 框架，根据 dva 最佳实践推荐，目录调整如下：

src          // 开发环境目录
|- components                // 展示组件目录
|    |- less                 // 样式资源目录
|    |- images               // 图片资源目录
|    |- json                 // 静态数据目录
|    |- doc                  // 使用文档目录
|    |- __test__             // 单元测试用例目录
|    |- 组件名（eg: Icon.js） // 组件源码文件
|    `- index.js             // 组件入口文件
|- routes                    // 【*】路由组件目录，作用同之前的容器组件目录（containers）
|- models                 // 【*】数据模型目录，作用同之前的异步流目录异步数据处理部分的代码（redux）
|- services               // 【*】服务目录，作用同之前的异步流目录调用接口部分的代码
|- utils                  // 工具集目录
|    `- axios.js          // 请求及全局异常处理配置文件
|- configs                // 配置文件目录
|    `- api.js            // API 接口地址配置对象
|- sprite                 // CSS Sprite 源图目录
.build        // 构建中转环境目录
dist          // 生产环境目录
    `- resources             // 资源目录
        |- dll               // 脚本资源目录
        |   `-vendor-xxxx.js
        |- js                // DLL 资源目录
        |  |- commons-xxxx.js
        |  `- index-xxxx.js
        |- css               // 样式资源目录
        `- images            // 图片资源目录
router.js      // 【*】单页应用入口组件，类似于之前的（App.js）
index.js       // 【*】单页应用入口文件，类似于之前的（index.js + store 目录）
index.ejs      // HTML 模板文件

项目类型为 “库” 时，生产环境目录结构如下：

library          // 生产环境（NPM 部署目录）
    |- es        // import { a } from 'library';
    |- lib       // var a = require('library/a');
    `- dist      // <script src="libraryName.js"></script>
